# 边江 RVC TTS 浏览器扩展 — 项目计划书

> **项目代号**: BianjiangVOC  
> **版本**: v1.0  
> **日期**: 2026-05-20  
> **目标**: 让 Edge 浏览器朗读任何网页内容时，听到的是边江的声音

---

## 一、项目背景

用户已在 Windows 上通过 RVC（Retrieval-based Voice Conversion）训练出高质量的边江声音模型（V2，第 23 轮），并验证了推理效果。现需开发一套端到端系统，将浏览器原生 TTS 输出实时替换为 RVC 音色转换后的音频。

## 二、技术架构

```
┌─────────────────────────────────────────────────┐
│                  Edge 浏览器                       │
│  ┌───────────────────────────────────────────┐  │
│  │           网页 (speechSynthesis API)         │  │
│  │               ↕  content script             │  │
│  │     ┌───────────────────────────────┐     │  │
│  │     │  Bianjiang VOC Extension       │     │  │
│  │     │  - 拦截 speak / cancel          │     │  │
│  │     │  - 发送文本到后端                │     │  │
│  │     │  - 播放返回的音频                │     │  │
│  │     └────────────┬──────────────┘     │  │
│  └──────────────────┼────────────────────┘  │
└─────────────────────┼───────────────────────┘
                      │ POST /synthesize
                      ▼
┌─────────────────────────────────────────────────┐
│              Python 后端 (FastAPI :8765)           │
│                                                   │
│  文本 → edge-tts → 临时音频 → RVC(v2) → 输出音频   │
│                                                   │
│  依赖: runtime\ (已有 torch + DML + faiss 环境)    │
│        + edge-tts 库                               │
└─────────────────────────────────────────────────┘
```

### 组件关系

| 组件 | 职责 | 关键技术 |
|------|------|----------|
| Content Script | 拦截 speechSynthesis API | Manifest V3, 原生 API Hook |
| Popup / Options | 用户控制界面 | HTML + JS, Chrome Extension API |
| 后端 API | 接收文本 → 返回转换音频 | FastAPI, asyncio |
| edge-tts 模块 | 生成标准 TTS 音频 | edge-tts (微软语音) |
| RVC 推理引擎 | 音色转换 | rtrvc.RVC + index 检索 |
| 配置文件 | 管理模型路径和参数 | YAML |

## 五、风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| edge-tts 调用失败 | 低 | 中 | fallback 到 pyttsx3 或直接返回错误 |
| RVC 推理超时（长文本） | 中 | 中 | 文本分片（每段 ≤200 字），结果拼接 |
| DML GPU 内存不足 | 低 | 高 | 设置 max_concurrent=1，串行处理 |
| 某些网站 CSP 阻止 content script | 中 | 低 | 不影响扩展注入，只影响 fetch；用 background script 代理请求 |
| Edge 更新后 API 行为变化 | 低 | 中 | 订阅 Chrome/Edge 更新日志，预留适配 |
