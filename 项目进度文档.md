# BianjiangRVC - 项目进度文档

## 一、项目概述

用 RVC（Retrieval-based Voice Conversion）模型将浏览器 TTS 朗读替换为「边江」的声音。用户已训练好 RVC V2 模型，通过本地后端 + 浏览器扩展实现：选中网页文字 → TTS 合成 → RVC 音色转换 → 播放。

**项目代号**: BianjiangVOC
**版本**: v1.0
**状态**: ✅ 已结项


### 数据处理流

```
文本输入
    ↓
edge-tts 源声音合成 (推荐: zh-CN-YunxiNeural)
    ↓ temp WAV
librosa 加载/重写为标准 WAV
    ↓ temp WAV
RVC 推理 (f0提取 + 特征检索 + 音色转换)
    ↓ numpy array (40000Hz)
sf.write → WAV 字节流
    ↓
浏览器 Audio 播放
```

## 五、文件详解

### 一键启动/关闭

| 文件 | 说明 |
|------|------|
| `一键启动.bat` | 检查端口 → 设置环境变量 → 启动后端 → 轮询 /health 直到就绪。首次启动需等待 ZLUDA 编译约 2-3 分钟 |

### 后端核心

| 文件 | 说明 |
|------|------|
| `backend/app.py` | FastAPI 服务入口。3 个路由：`GET /health`、`GET /models`、`POST /synthesize`。启动时加载 RVC 模型为全局单例，含 CORS 中间件 |
| `backend/rvc_engine.py` | RVC 推理引擎类 `R`。核心方法 `synth(text, voice, f0_up_key, model)`：调用 edge-tts 生成源音频 → librosa 标准化 → VC.vc_single 执行音色转换 → 返回 (sr, audio)。线程安全（threading.Lock） |
| `backend/config.yaml` | YAML 配置。模型路径、权重目录、index 路径、推理参数（f0_up_key/f0_method/index_rate 等）、服务端口 |


## 七、推理参数（已验证最佳值）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| f0_up_key | 0 | 变调（半音），范围 -12 ~ +12 |
| f0_method | pm | 基频提取算法 (pm/harvest/crepe/rmvpe/fcpe) |
| index_rate | 0.75 | 特征检索混合比例，0=纯RVC，1=纯index |
| filter_radius | 3 | 中值滤波半径 |
| rms_mix_rate | 0.25 | 源音频 RMS 混合率 |
| protect | 0.33 | 清辅音保护 (0~1) |
| resample_sr | 0 | 重采样率，0=不重采样 |

## 八、推理管线 (rvc_engine.py)

```
R.__init__(config)
  ├── 读取 config.yaml
  ├── os.chdir(rvc_root) → 进入 RVC 仓库目录
  ├── import zluda → 初始化 ZLUDA（AMD GPU CUDA 模拟层）
  ├── load_dotenv(.env) → 加载 RVC 环境变量
  └── _lm(model_name) → 加载模型
       ├── Config() → RVC 全局配置（含设备检测）
       └── VC(config).get_vc(model) → 加载模型权重 + index

R.synth(text, voice, f0_up_key, model)  [async]
  ├── 如果 model 变化 → _lm(new_model) 热切换
  ├── _tts_edge(text, voice) → await edge_tts.Communicate.save()
  │     返回临时 WAV 文件路径
  ├── _wav(wav_path) → librosa.load + sf.write（标准化 WAV 格式）
  │     返回标准化后的临时 WAV 路径
  ├── VC.vc_single(0, wav, f0_up_key, None, f0_method, index_path, ...)
  │     → 基频提取 → 特征检索 → 音色转换
  │     返回 (info_string, (sr, audio_numpy))
  └── 清理临时文件 → 返回 (sr, audio)
```

## 十、已知限制

| 限制 | 说明 |
|------|------|
| ZLUDA 首次编译 | 首次启动需 2-3 分钟编译 ZLUDA 内核，之后秒级 |
| 模型训练收敛 | 模型与底模差异约 0.2%，训练未充分收敛。建议在 Linux/ROCm 下重训 |
| 源 TTS 依赖网络 | edge-tts 需要访问微软 TTS 服务器，断网无法使用 |
| 单次推理串行 | 使用 threading.Lock 保证串行推理，多请求排队 |
| 长文本分片 | 当前未实现自动分片，超长文本可能超时 |
